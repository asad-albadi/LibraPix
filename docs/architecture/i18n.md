# i18n Architecture

Internationalization is first-class from day one.

## Baseline decisions

- All user-facing text uses keys (`TextKey`) rather than scattered literals.
- Locale lookup is centralized in `librapix-i18n`.
- Default/fallback locale is `en-US`.
- UI crates request strings through the translator API.
- Locale preference is loaded from `librapix-config` during app bootstrap.
- Projection/search validation labels are keyed and routed through `TextKey` values.

## Extension path

- Add additional locale catalogs without touching UI call sites.
- Introduce runtime locale switching via application settings state.
- Keep fallback behavior deterministic when a key is missing.
